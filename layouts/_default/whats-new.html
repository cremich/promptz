After running promptz for almost a year, I learned that the browser-based contribution introduced friction in your workflows. The community wanted to contribute content using familiar git-based workflows rather than through a web interface.

The key goal of v2: Improve your developer experience and reduce friction in contribution, collaboration, and usage by embracing the tools and workflows you already know and love.

All prompts, project rules, and custom agents are now managed as markdown files in the promptz repository, making it easier than ever to contribute, collaborate, and maintain high-quality content for the Amazon Q Developer community.

Git-Based Workflow

No more browser forms! Contribute using familiar git workflows with markdown files and pull requests.

  • Fork repository on GitHub
  • Create markdown files with frontmatter
  • Automated validation & approval

Better Performance

Static site generation delivers blazing-fast loading times and enhanced search capabilities.

  • Static site generation
  • Optimized search
  • Enhanced content discovery
